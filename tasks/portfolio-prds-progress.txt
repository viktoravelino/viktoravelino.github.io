## Codebase Patterns
- **Layout**: Use `main.astro` layout wrapping all pages; it includes global CSS, meta tags, and `class="dark"` on `<html>`
- **Fonts**: Google Fonts loaded in `main.astro` head — Instrument Serif (display/headings) + Geist (body text). Applied via inline `style` attribute on elements.
- **Astro components for static sections**: Hero section is `.astro` since it's static content. React components only needed for interactive parts (e.g., SocialLinks needs `client:load` for lucide-react icons).
- **Data files**: Keep section data in `src/data/<section>.ts` with typed exports. Import into both Astro and React components.
- **CSS animations**: Hero uses CSS keyframe animations with `--delay` CSS variable for stagger (`hero-fade-in` class). Define keyframes + utility classes in `global.css`.
- **shadcn Button `asChild`**: Use `asChild` prop with `<a>` tags for link buttons — avoids nesting `<button>` inside `<a>`.
- **Tailwind v4**: Uses `@theme inline` block in global.css for custom theme values. No `tailwind.config.js` — everything is CSS-based.
- **Build check**: Always run `npm run build` to verify — no linter/tests configured yet.
